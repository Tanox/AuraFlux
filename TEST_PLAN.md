# AuraFlux 测试计划

## 1. 测试范围

### 1.1 核心功能模块
- **音频处理模块**：文件播放、麦克风输入、音频分析
- **可视化渲染模块**：2D/3D 可视化效果、AI 背景生成
- **AI 集成模块**：Gemini API 调用、音频识别、视觉配置生成
- **状态管理模块**：应用状态、视觉状态、AI 状态
- **用户界面模块**：控制面板、设置面板、响应式设计
- **国际化模块**：多语言支持、翻译文件验证
- **性能监控模块**：FPS 监控、设备性能检测

### 1.2 测试类型
- **单元测试**：验证单个函数和组件的功能
- **集成测试**：验证模块间的交互
- **系统测试**：验证整体功能和用户体验
- **性能测试**：评估应用性能和响应速度
- **国际化测试**：验证多语言支持的正确性

## 2. 测试策略

### 2.1 单元测试
- 使用 Jest + React Testing Library
- 测试覆盖率目标：≥80%
- 重点测试：
  - 核心工具函数
  - 自定义钩子
  - 服务层逻辑
  - 状态管理逻辑

### 2.2 集成测试
- 测试模块间的交互
- 模拟 API 调用和外部依赖
- 验证数据流和状态传递

### 2.3 系统测试
- 端到端测试用户流程
- 验证完整的功能链路
- 测试边界条件和异常场景

### 2.4 性能测试
- 运行性能测试脚本
- 监控 FPS 和资源使用
- 评估不同设备上的表现

### 2.5 国际化测试
- 验证所有语言的翻译完整性
- 测试语言切换功能
- 检查文本布局和显示

## 5. 测试执行计划

### 5.1 测试阶段
1. **单元测试阶段**：验证核心功能模块
2. **集成测试阶段**：验证模块间交互
3. **系统测试阶段**：验证整体功能
4. **性能测试阶段**：评估应用性能
5. **国际化测试阶段**：验证多语言支持

### 5.2 测试执行流程
1. 运行单元测试：`pnpm test`
2. 运行测试覆盖率：`pnpm test:coverage`
3. 运行性能测试：`./test/performance-test.ps1`
4. 运行国际化测试：`ts-node test/test-i18n.ts`
5. 手动系统测试：验证完整用户流程
