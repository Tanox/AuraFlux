## 1. 常见问题 (FAQ)

### Q1: 为什么我看不到任何可视化效果？
**A:** 请检查以下几点：
1. 确保您的浏览器 **麦克风权限** 已开启。
2. 确保您的设备正在播放音频，或者麦克风可以接收到声音。
3. 尝试刷新页面。
4. 检查 **Sensitivity** (敏感度) 设置是否过低。

### Q2: AI 识别功能总是失败，怎么办？
**A:** 可能的原因包括：
1. **API Key 问题**: 检查环境变量 `GEMINI_API_KEY` 是否正确配置且有效（服务端）。
2. **网络问题**: 确保您的网络可以访问 Google Gemini API 服务。
3. **音频问题**: 环境噪音大可能影响识别，请尝试靠近音源。

### Q3: 页面运行卡顿怎么办？
**A:** WebGL 渲染对性能有一定要求。
1. 在 **Settings** 面板中，降低 **Quality** (质量) 设置为 **Low**。
2. 确保浏览器已开启硬件加速。
3. 关闭其他占用大量系统资源的程序。

### Q4: 移动设备支持如何？
**A:** 支持现代移动浏览器 (Chrome, Safari)。但由于移动设备 GPU 性能限制，建议使用 **2D 模式** (如 Waveform, Bars) 或降低渲染设置。iOS Safari 需要用户手动开启麦克风权限（点击页面任意位置）。

## 2. 故障排除指南

### 2.1 麦克风权限被拒绝
**现象**: 提示 "Microphone access denied"。
**解决**:
1. 点击浏览器地址栏左侧的 "锁" 图标。
2. 找到 "麦克风" 选项，设置为 "允许" (Allow)。
3. 刷新页面。

### 2.2 AI 服务响应 403/401 错误
**现象**: 控制台报错 `POST ... 403 Forbidden`。
**解决**:
1. 确认 API Key 有效。
2. 确认 API Key 绑定的 Google Cloud 项目已启用 "Generative Language API"。

### 2.3 3D 场景黑屏
**现象**: UI 显示正常，但背景全黑。
**解决**:
1. 打开浏览器开发者工具 (F12) -> Console。
2. 查看是否有 WebGL 相关错误 (e.g., "WebGL context lost")。
3. 更新显卡驱动，或重启浏览器。

### 2.4 构建失败
**现象**: `npm run build` 失败。
**解决**:
1. 检查 `package.json` 依赖是否安装完整 (`npm install`)。
2. 检查 TypeScript 类型错误 (`npm run lint`)。
3. 清理 Next.js 缓存 (`rm -rf .next`) 后重试。

### 2.5 录制功能无法使用
**现象**: 点击录制按钮无反应。
**解决**:
1. 检查浏览器是否支持 MediaRecorder API。
2. 确保存储空间充足。
3. 尝试使用 Chrome 浏览器。

### 2.6 音频文件无法加载
**现象**: 选择本地音频文件后无反应。
**解决**:
1. 确保文件格式为浏览器支持的格式 (MP3, WAV, OGG)。
2. 检查文件大小是否过大。
3. 尝试使用不同的音频文件。

### 2.7 可视化模式切换无反应
**现象**: 切换可视化模式后效果不变。
**解决**:
1. 尝试刷新页面。
2. 检查浏览器控制台是否有错误。
3. 确保您的浏览器支持 WebGL 2.0。

## 5. 最佳实践

### 5.1 性能优化
- 使用 Chrome 浏览器获得最佳体验
- 对于低配置设备，选择 2D 可视化模式
- 关闭不必要的浏览器扩展
- 定期清理浏览器缓存

### 5.2 安全建议
- 不要在公共网络上使用敏感的 API Key
- 定期更新浏览器到最新版本
- 注意保护个人隐私，避免在公共场合使用麦克风

### 5.3 常见操作技巧
- 使用快捷键提高操作效率
- 保存常用设置为预设
- 使用录制功能分享您的创作
- 尝试不同的音频源获得不同的可视化效果
