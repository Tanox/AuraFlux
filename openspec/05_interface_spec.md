# OpenSpec: UI/UX 与交互规范 (05)

## 1. 媒体库交互优化
- **双击播放:** 为了防止误触，曲目列表由“单击播放”升级为“双击播放”。
- **动态删除:** 条目右侧的 'X' 删除按钮仅在鼠标悬停或元素获得焦点时显示。

## 2. 布局系统
- **Bento Grid:** 设置界面采用模块化 Bento 布局。
- **全局拖拽:** 支持任意界面直接拖入音频文件进行导入，具备高优先级 z-index 的上传引导层。
- **BottomBar 收窄:** 底部快速控制栏宽度优化为 `max-w-lg`，提升大屏幕下的视觉平衡感。

## 3. 图层控制 (Overlay Control) - v1.8.83 更新
- **紧凑布局:** 移除了多余的排版预览小部件，将“文字输入区”紧贴“内容源选择器”并置，实现了更短的输入框宽度。
- **多功能内容源:** 支持 AI 智能 HUD（自动识别）、手动文本、歌曲元数据以及实时时钟。
- **锚点选择:** 在主设置区侧边集成 3x3 锚点选择器，通过物理位置映射进行操作。

## 4. 开关组件规范 (Toggle Standards) - v1.8.84 引入
- **物理结构:** 开关按钮 (Switch) 必须位于左侧，标签文字 (Label) 位于右侧。
- **排版对齐:** 文字标签必须保持 `text-left`（向左对齐），并与开关保持统一的间距 (`gap-3`)。
- **视觉一致性:** 在 `VisualizerPreview` 等列表项中，包含状态开关也必须移至条目左侧，以维持全站交互流的一致性。

## 5. VJ 控制控制台
- **热键系统:** 1-6 切换面板, R 随机, F 全屏, Space 暂停/播放。
- **图标更新:** 关闭按钮更新为向下箭头 (Chevron Down)，提升视觉直觉感。
