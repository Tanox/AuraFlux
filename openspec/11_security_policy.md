# OpenSpec: 安全策略与漏洞报告 (11)

## 1. 核心安全理念 (v1.9.36)
Aura Flux 的设计将用户隐私和数据安全置于首位。我们遵循“本地优先”和“最小化数据”原则。

- **本地优先处理 (Local-First Processing):** 核心的音频频谱分析、视觉渲染和录制功能完全在用户的浏览器中本地执行。敏感的音频数据绝不会未经处理就离开您的设备。
- **最小化数据上报 (Minimal Data to Cloud):**
    - **AI 歌曲识别:** 仅将一个短暂（约 4 秒）且无法识别个人身份的音频片段发送至 Google Gemini API 进行分析。此请求不包含任何用户身份信息、IP 地址或其他元数据。
    - **API 密钥:** 用户提供的 Gemini API 密钥仅存储在浏览器的 `LocalStorage` 中，并经过基础的 Base64 编码，绝不会上传到我们的服务器（因为我们没有服务器）。

## 2. 支持的版本
我们仅为最新发布的版本提供安全更新。请确保您始终运行最新版本的 Aura Flux。

| 版本    | 支持状态 |
| ------- | -------- |
| `1.9.x` | ✅ 支持   |
| `< 1.9` | ❌ 不支持 |

## 3. 报告安全漏洞
我们非常重视社区发现的安全问题。如果您在 Aura Flux 中发现了一个安全漏洞，请负责任地向我们报告。

- **报告渠道:** 请通过电子邮件将漏洞详情发送至 `security@ewuse.com`。
- **报告内容:** 请在报告中提供以下信息：
    1.  漏洞的详细描述，包括其潜在影响。
    2.  重现漏洞所需的步骤、代码片段或概念验证。
    3.  您使用的浏览器和操作系统版本。
- **处理流程:**
    1.  我们将在 48 小时内确认收到您的报告。
    2.  我们将对漏洞进行评估，并确定其严重性。
    3.  我们将开发修复程序，并在下一个版本中发布。
    4.  在修复程序发布后，我们将公开感谢您的贡献（如果您同意）。

**请不要在公开的 GitHub Issues 中报告安全漏洞。**
