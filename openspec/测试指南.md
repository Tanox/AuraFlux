# AuraFlux 测试指南

## 测试结构

项目的测试文件按照功能模块组织，位于 `test/` 目录下：

```
test/
├── hooks/
│   ├── audio/         # 音频处理相关测试
│   └── state/         # 状态管理相关测试
├── services/         # 服务层测试
├── utils/            # 工具函数测试
└── test-i18n.ts      # 国际化测试
```

## 测试框架

- **测试运行器**：Jest
- **测试库**：@testing-library/react
- **测试环境**：jsdom (模拟浏览器环境)

## 运行测试

### 运行所有测试
```bash
pnpm test
```

### 运行测试并生成覆盖率报告
```bash
pnpm test:coverage
```

### 监视模式运行测试
```bash
pnpm test:watch
```

## 编写测试的最佳实践

### 1. 测试结构

每个测试文件应遵循以下结构：

```typescript
// File: test/path/to/module.test.tsx

import { renderHook, act } from '@testing-library/react';
import { useModule } from '@/path/to/module';

// Mock 依赖项
jest.mock('@/path/to/dependency');

// 模拟全局对象（如需要）
Object.defineProperty(window, 'someGlobal', {
  value: mockValue,
  writable: true,
});

describe('useModule', () => {
  beforeEach(() => {
    jest.clearAllMocks();
  });

  test('should initialize with default state', () => {
    // 测试逻辑
  });

  test('should handle specific case', () => {
    // 测试逻辑
  });
});
```

### 2. 模拟依赖

- **外部依赖**：使用 `jest.mock()` 模拟
- **浏览器 API**：使用 `Object.defineProperty` 模拟
- **组件依赖**：使用 `jest.fn()` 创建模拟函数

### 3. 测试覆盖重点

- **核心功能**：确保所有核心功能都有测试覆盖
- **边缘情况**：测试空输入、错误输入等边缘情况
- **状态管理**：测试状态的初始化、更新和持久化
- **错误处理**：测试错误处理逻辑

### 4. 测试命名规范

- 测试文件名：`[模块名].test.tsx` 或 `[模块名].test.ts`
- 测试用例：使用描述性的名称，如 `should initialize with default state`

### 5. 测试性能

- 避免在测试中进行网络请求
- 合理使用 `beforeEach` 和 `afterEach` 清理测试环境
- 避免测试中使用真实的 DOM 操作（使用 jsdom 模拟）

## 常见测试问题及解决方案

### 1. JSX 语法错误

**问题**：测试运行时出现 "Support for the experimental syntax 'jsx' isn't currently enabled"

**解决方案**：确保项目根目录有 `.babelrc` 文件，包含 React 预设：

```json
{
  "presets": [
    "@babel/preset-env",
    "@babel/preset-react",
    "@babel/preset-typescript"
  ]
}
```

### 2. 模拟浏览器 API

**问题**：测试中使用了浏览器特有的 API（如 `localStorage`、`navigator.mediaDevices`）

**解决方案**：在测试文件中模拟这些 API：

```typescript
// 模拟 localStorage
const mockLocalStorage: Record<string, string> = {};
Object.defineProperty(window, 'localStorage', {
  value: {
    getItem: jest.fn((key) => mockLocalStorage[key] || null),
    setItem: jest.fn((key, value) => {
      mockLocalStorage[key] = value;
    }),
    removeItem: jest.fn((key) => {
      delete mockLocalStorage[key];
    }),
    clear: jest.fn(() => {
      Object.keys(mockLocalStorage).forEach(key => delete mockLocalStorage[key]);
    }),
  },
  writable: true,
});
```

### 3. 异步测试

**问题**：异步操作的测试失败

**解决方案**：使用 `async/await` 和 `act` 包裹异步操作：

```typescript
test('should handle async operation', async () => {
  const { result, waitFor } = renderHook(() => useAsyncHook());

  await act(async () => {
    await result.current.asyncFunction();
  });

  await waitFor(() => {
    expect(result.current.data).toBeDefined();
  });
});
```
