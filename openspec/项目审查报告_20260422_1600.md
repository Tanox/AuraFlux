## 1. 项目结构分析

### 1.1 目录结构
项目采用标准的 Next.js 14 应用结构，目录组织合理，主要包含以下核心目录：

- `src/` - 源代码目录
  - `app/` - Next.js App Router 应用目录
  - `components/` - React 组件
  - `context/` - 状态管理上下文
  - `hooks/` - 自定义 React Hooks
  - `locales/` - 国际化语言文件
  - `services/` - 服务层
  - `types/` - TypeScript 类型定义
  - `utils/` - 工具函数
- `openspec/` - 项目规范和文档
- `public/` - 静态资源
- `scripts/` - 脚本文件

### 1.2 代码组织
- **组件化设计**: 采用模块化组件设计，将 UI 拆分为多个可复用组件
- **状态管理**: 使用 React Context API 进行全局状态管理，结构清晰
- **类型安全**: 全面使用 TypeScript 类型定义，提高代码可维护性
- **国际化支持**: 完善的 i18n 实现，支持多语言切换

## 2. 代码质量分析

## 3. 功能模块审查

### 3.1 音频引擎
- **音频分析**: 基于 Web Audio API 的实时 FFT 分析
- **多源支持**: 支持麦克风、本地文件和 URL 三种音频源
- **播放控制**: 完整的播放控制功能，包括播放、暂停、跳过等
- **设备管理**: 支持音频设备切换和管理

### 3.2 视觉渲染系统
- **混合渲染**: 支持 2D Canvas 和 Three.js WebGL 渲染
- **多种模式**: 实现了多种可视化模式，包括数字网格、丝绸波浪、海洋波浪等
- **颜色管理**: 支持自定义颜色主题和自动颜色循环
- **性能适配**: 根据设备性能自动调整渲染复杂度

### 3.3 AI 集成
- **歌曲识别**: 使用 Gemini 3.0 进行实时歌曲识别
- **歌词分析**: 支持歌词检索和显示
- **情绪分析**: 基于音乐情绪自动调整视觉效果
- **AI 视觉导演**: 自动优化视觉设置以匹配音乐风格

### 3.4 用户界面
- **控制面板**: 响应式控制面板，支持展开/收起
- **覆盖层**: 歌曲信息、歌词和自定义文本覆盖层
- **设置管理**: 完整的设置管理，支持预设保存和加载
- **手势支持**: 支持移动设备手势操作
