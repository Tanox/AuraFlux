# 迁移指南

## 版本信息
- **版本**: v2.3.5
- **更新日期**: 2026-04-21
- **作者**: Sut

## 1. 版本迁移指南

### 1.1 从v1.x.x 迁移到v2.0.x

#### 1.1.1 核心架构变更
- **应用入口**: 从 `src/App.tsx` 迁移到 `src/components/App.tsx`
- **状态管理**: 从多个独立的 context 迁移到统一的 `AppContext`
- **文件结构**: 采用更清晰的功能模块化结构

#### 1.1.2 依赖变更
- **React**: 升级到 React 19
- **Three.js**: 升级到最新版本
- **Google Gemini**: 从 v1 升级到 v3

#### 1.1.3 API 变更
- **音频 API**: 从 `useAudioContext` 迁移到 `useAudio` Hook
- **AI API**: 从旧的 `aiService` 迁移到新的增强版 `aiService`

### 1.2 从v2.3.2 迁移到v2.3.4

#### 1.2.1 文档结构变更
- **文档位置**: 从 `openspec/` 目录迁移到 `spec/` 目录
- **文档结构**: 采用更清晰的分类和层级结构
- **文档内容**: 增加了技术细节和代码示例

#### 1.2.2 功能变更
- **视觉模式**: 移除了 RINGS 和 PARTICLES 模式，添加了 STARFIELD 模式
- **音频处理**: 优化了音频分析算法
- **AI 集成**: 增强了 AI 服务的可用性检查

## 2. 迁移步骤

### 2.1 代码迁移

#### 2.1.1 步骤 1: 备份现有代码
```bash
# 备份现有代码
cp -r src/ src_backup/
```

#### 2.1.2 步骤 2: 更新依赖
```bash
# 更新依赖
pnpm install
```

#### 2.1.3 步骤 3: 更新文件结构
- 根据新的文件结构重新组织代码
- 更新导入路径，使用 `@/` 别名

#### 2.1.4 步骤 4: 更新 API 调用
- 替换旧的 API 调用为新的 API
- 更新状态管理方式

### 2.2 配置迁移

#### 2.2.1 环境变量
- 确保设置了 `GEMINI_API_KEY` 环境变量（服务端）

#### 2.2.2 存储迁移
- 新旧版本的 localStorage 数据会自动兼容
- 数据结构保持向后兼容

## 3. 常见问题

### 3.1 迁移过程中的常见错误

#### 3.1.1 导入路径错误
- **错误**: `Module not found: Can't resolve '../types'`
- **解决方案**: 使用完整的导入路径，例如 `../types/index.ts`

#### 3.1.2 依赖冲突
- **错误**: `Peer dependency conflict`
- **解决方案**: 运行 `pnpm dedupe` 解决依赖冲突

#### 3.1.3 API 密钥错误
- **错误**: `AI service is not available`
- **解决方案**: 确保设置了正确的 `GEMINI_API_KEY` 环境变量（服务端）

### 3.2 性能问题

#### 3.2.1 帧率下降
- **问题**: 可视化效果帧率下降
- **解决方案**: 降低视觉质量设置或使用更简单的可视化模式

#### 3.2.2 内存使用过高
- **问题**: 应用内存使用过高
- **解决方案**: 关闭不需要的视觉效果，清理播放列表

## 4. 回滚策略

### 4.1 如何回滚到旧版本

#### 4.1.1 代码回滚
```bash
# 恢复备份
rm -rf src/
mv src_backup/ src/
```

#### 4.1.2 依赖回滚
```bash
# 安装旧版本依赖
pnpm install --frozen-lockfile
```

### 4.2 数据恢复
- **localStorage**: 数据会自动保留
- **IndexedDB**: 数据会自动保留
