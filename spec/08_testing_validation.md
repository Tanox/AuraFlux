## 1. 测试层级

### 1.1 单元测试
- **重点**: 逻辑验证
- **目标文件**:
  - `audioUtils.ts` - WAV 转换功能
  - `beatDetector.ts` - 节拍检测算法
  - 工具函数和辅助方法

**测试策略:**
- 测试函数边界条件
- 测试异常处理
- 测试性能和准确性

### 1.2 集成测试
- **重点**: 流程集成
- **测试场景**:
  - `useAudio` 到 `VisualizerCanvas` 的数据传递
  - AI 服务集成
  - 状态管理流程

**测试策略:**
- 使用 `MOCK` 模式测试 UI 响应
- 测试组件间数据流
- 测试状态同步

### 1.3 端到端测试
- **重点**: 完整用户流程
- **关键场景**:
  1. 麦克风授权 → 视觉反馈
  2. 音频文件导入 → 播放控制
  3. 歌词加载 → 显示效果
  4. 视频录制 → 导出功能

**测试策略:**
- 使用 Playwright 进行自动化测试
- 模拟用户交互
- 验证完整功能流程

## 2. 性能基准

### 2.1 帧率要求
- **标准**: 在标准硬件（Intel/M1+ 集显）上
- **要求**: Med 模式下稳定在 60FPS

### 2.2 性能监控
- **工具**: FPSCounter 组件
- **指标**:
  - 渲染帧率
  - 内存使用
  - CPU 占用
  - 音频处理延迟

### 2.3 Worker 负载
- **要求**: 2D 渲染不应阻塞 UI 线程
- **优化**: 使用 Web Workers 处理密集计算

## 4. 测试工具

### 4.1 测试框架
- **推荐**: Jest
- **配置**: 支持 TypeScript

**代码示例:**
```javascript
// jest.config.js 核心结构
// File: jest.config.js | Version: v2.2.25
module.exports = {
  preset: 'ts-jest',
  testEnvironment: 'jsdom',
  setupFilesAfterEnv: ['<rootDir>/src/setupTests.ts'],
  moduleNameMapper: {
    '^@/(.*)$': '<rootDir>/src/$1'
  },
  collectCoverageFrom: [
    'src/**/*.{ts,tsx}',
    '!src/**/*.d.ts',
    '!src/**/index.ts',
    '!src/**/types/**'
  ],
  coverageThreshold: {
    global: {
      branches: 80,
      functions: 80,
      lines: 80,
      statements: 80
    }
  }
};
```
